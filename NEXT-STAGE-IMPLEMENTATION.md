## Operating Rule

The Drive copy is now the practical source of truth for organizational review. Open START HERE first for the current launch-gate review entrypoint, use the Launch Gates tab as the single weekly control view for the six open gates, use Owner Evidence Review for source-backed owner nominations before any names move into Leader Roster, use Owner Appointment Routing to nominate candidates and confirm backup coverage before Leader Roster entry, use the Owner Appointment Confirmation Templates Doc to capture candidate, backup, authority and evidence-routing details before `Leader Roster` changes, use Governance Authority Review to confirm final decision rights and escalation paths before HR-005 or DEC-003 can close, use Human Review Routing to assign reviewers and target dates before any HR-001 through HR-007 sign-off is entered, use the Human Review Sign-off Templates Doc to capture qualified reviewer wording and evidence-routing before `Human Review` outcomes change, use Pilot Cycle Plan to schedule and track one leadership-team pilot before GATE-003 can close, use the Pilot Evidence Capture Templates Doc to capture non-sensitive observations, debrief summaries and accepted-change routing before pilot findings update trackers, use Gate Evidence Checklist to collect proof before any Launch Gates status moves to complete, use Leadership Review Schedule to set meeting dates and outputs for the open gates, use Leadership Review Agenda to run the scheduled reviews with explicit prompts and decision boundaries, use Leadership Review Outputs to record meeting outputs and route follow-up updates after scheduled reviews, use Leadership Review Communications and the Leadership Review Message Templates Doc to draft, send and follow up role-only outreach without storing contact details or treating outreach as approval, use the Restricted Data Placement Approval Templates Doc to capture accountable data-owner approval of storage location, access group, field purpose and retention before DEC-005/TASK-014 updates, use Website Content Review to classify proposed public/protected/restricted/withheld pages, resources, videos, dates and dashboards before any production build, use the Website Access Model Decision Templates Doc with Website Access Review to capture the selected model and WAR-001 through WAR-014 area decisions before DEC-004/GATE-005 can close, use the Launch Gate Review Pack to run a single leadership session across GATE-001 through GATE-006 before any gate status changes, use the Production Website Publication Approval Templates Doc to capture a final GATE-006/DEC-006/TASK-016 go/no-go only after access, authority, copyright, content, restricted-data and technical evidence is accepted, and treat the local/Drive HTML START HERE, Planning, Templates, Gate Review and Access views as synthetic prototype surfaces only. The local repository remains the audit trail, template source and recovery package. Do not publish the production website or broaden sharing until the live tracker shows the relevant approvals.
