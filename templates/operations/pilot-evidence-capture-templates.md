# Pilot Evidence Capture Templates

Use this guide with the native Google Doc: https://docs.google.com/document/d/1F3rh9hJd9FPXFAAHsrVnVeeH-3P8FilO6_9en3Z97W4/edit

This package prepares evidence capture for GATE-003. It does not run the pilot, prove that a pilot happened, approve pilot findings, change any tracker status, authorize restricted-data migration, approve website access or authorize publication.

## When To Use

Use the capture templates during and immediately after one real Journey leadership-team pilot. Keep the Pilot Log as the structured source of truth and use this pack for consistent observation wording, debrief summaries and accepted-change routing.

Do not enter participant names, medical information, financial records, private contact details, pastoral disclosures, private reviewer details or unapproved source material in the shared Pilot Log or broadly shared Docs.

## Shared Observation Record Fields

- Pilot item ID
- Cycle or session tested
- Workflow tested
- Observer role
- Observation type: unclear step / duplicated work / missing resource / unnecessary field / access issue / safety or data concern / decision needed / improvement
- What happened
- Why it matters
- Affected resource or tracker
- Suggested change
- Severity: low / medium / high / blocker
- Accountable owner role
- Status: logged / triaged / change approved / change made / deferred / blocked
- Evidence link or note
- Sensitive data check

## Wording Blocks

### Unclear Step

The team could not tell the next action because [resource, field, or instruction] did not make [decision or sequence] clear. Proposed fix: [specific wording, link, owner role, or checklist change].

### Duplicated Work

The same information had to be entered or checked in [location one] and [location two]. Proposed fix: choose [single source of truth] and link or reference it from the other location.

### Missing Resource

The team expected [template, guide, checklist, policy, meeting artifact, or decision record] but could not find it from [start point]. Proposed fix: create or link the resource from [tracker, index, or pack].

### Unnecessary Field

The field [field name] created friction because [reason] and did not change an operating decision. Proposed fix: remove it, make it optional, or replace it with [simpler field].

### Access Issue

The team could not determine who should see [resource or website area]. Proposed fix: route the decision through Website Access Review or the accountable data owner before sharing changes.

### Safety Or Data Concern

The workflow risked exposing [category only] in a broad location. Proposed fix: move the detail to an approved restricted location and keep only role-based status in shared trackers.

### Decision Needed

Work could not continue until [qualified reviewer, programme authority, data owner, or website approver] decides [specific decision]. Proposed fix: assign the decision owner and date in the correct review surface.

### Improvement

The workflow worked, but [small change] would reduce time or confusion. Proposed fix: apply after owner approval and record the validation evidence.

## Debrief Summary Template

- Pilot date
- Facilitator role
- Workflows tested
- Attendees by role only
- What was easy to use
- What was confusing
- Duplicated work found
- Missing resources found
- Fields to remove or simplify
- Risks or blockers found
- Access or data decisions needed
- Changes accepted now
- Changes deferred
- Restricted follow-ups
- Next pilot or launch-gate decision

## Accepted Change Routing Template

- Change ID
- Originating pilot observation ID
- Approved by role and name
- Approval date
- Affected artifact
- Change summary
- Required updates
- Validation evidence
- Launch-gate impact
- Restricted-data impact
- Final status

## Tracker Update Rule

Record observations in `Pilot Log` first. Update `Pilot Cycle Plan`, `Gate Evidence Checklist`, `TASK-013`, `Event Readiness` and `Launch Gates` only when real pilot evidence exists and the accountable owner accepts the debrief outcome. Do not mark GATE-003 complete from this template alone.
