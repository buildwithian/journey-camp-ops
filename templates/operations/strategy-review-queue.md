# Strategy Review Queue

## Purpose

Use this queue to keep the six current Journey V3 strategy references owned, reviewed and aligned before production website build work, Drive maintenance changes, access-model changes or publication decisions.

## Current Live-state Note

The live `Strategy Review Queue` tab is populated with the six rows from [strategy-review-queue.csv](strategy-review-queue.csv), linked into Dashboard row 26, Instructions row 34 and Resource Library Index quality checks. Dashboard row 26 currently shows six pending strategy reviews.

## Privacy And Approval Rule

Keep `Next review` and any future decision evidence blank until the accountable reviewer records a real review output. Do not store private names, contact details, unpublished reviewer notes, restricted data, credentials, participant details, medical information or financial information in this queue.

## Status Values

- Review required
- Ready for review
- In progress
- Accepted for build-scope input
- Changes required
- Deferred
- Not approved / withheld

## Queue Coverage

The CSV template contains six strategy-review rows:

- SRQ-001 for the Alpha-Inspired Hub Target State.
- SRQ-002 for the Alpha-Style Website Benchmark And Adaptation Brief.
- SRQ-003 for Website Operating Hub Requirements.
- SRQ-004 for the Drive Copy Build Runbook.
- SRQ-005 for the Google Drive Consolidation And Resource Library Plan.
- SRQ-006 for the Google Drive Import Manifest.

## Operating Sequence

1. Review the Strategy Control Board prototype view and confirm each strategy reference is still current.
2. Use this queue to assign owner roles, reviewing authorities and next-review timing before updating website build scope or Drive maintenance routines.
3. Move a row to `Accepted for build-scope input` only after the named reviewing authority records evidence in the appropriate decision, review or handoff surface.
4. Keep Drive and Sheets as the source of truth; the prototype only mirrors the review path.
5. Route access, copyright, restricted-data, build and publication questions back to their dedicated tracker rows and approval templates.

## Non-Approval Boundary

This queue does not approve Alpha asset use, copyright permission, website access, build start, deployment, publication, Drive sharing, restricted-data movement, source deletion, DEC-006/TASK-016 updates or launch-gate closure. It makes strategy review and maintenance easier to manage before accountable humans approve downstream changes.
