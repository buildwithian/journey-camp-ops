# Template Improvement Queue

## Purpose

Use this queue to turn duplicate, rough or source-derived Journey V3 materials into summarized, consolidated and improved template outputs without overwriting source material or treating draft improvements as approved.

## Current Live-state Note

The live `Template Improvement Queue` tab is populated with the twelve rows from [template-improvement-queue.csv](template-improvement-queue.csv), linked into Resource Library Dashboard row 12, Instructions row 10 and Resource Library Index quality checks. Dashboard row 12 currently shows twelve template improvements pending.

## Privacy And Approval Rule

Keep all outputs review-required until the listed reviewer or approval authority accepts the improved template. Do not import private source content, real names, contact details, participant details, medical information, financial information, consent records or private photographs into the queue.

## Status Values

- Review required
- Draft improvement
- Ready for review
- Accepted as canonical
- Changes required
- Deferred
- Withheld

## Output Format Values

- Google Doc
- Google Sheet
- Folder-backed pack
- Website module
- Mixed Doc and Sheet

## Queue Coverage

The CSV template contains twelve improvement rows:

- TIQ-001 for start and launch routing.
- TIQ-002 for roles, RACI and owner appointment.
- TIQ-003 for leader training and onboarding.
- TIQ-004 for participant onboarding and registration.
- TIQ-005 for weekly sessions and facilitation.
- TIQ-006 for camps, events and rite-of-passage packs.
- TIQ-007 for administration trackers.
- TIQ-008 for finance and payment templates.
- TIQ-009 for logistics and equipment.
- TIQ-010 for safety, safeguarding and medical readiness.
- TIQ-011 for communications templates.
- TIQ-012 for quality, impact and post-event review.

## Operating Sequence

1. Confirm the related duplicate-resolution row or source bundle before drafting an improved output.
2. Draft the summarized template in the listed format using synthetic examples only.
3. Move the row to `Ready for review` only when the improved output has an owner, reviewer and target Drive or website placement.
4. Move the row to `Accepted as canonical` only after the listed reviewer or approval authority records acceptance.
5. Update Resource Library links and the operating hub only after acceptance, never from draft status alone.

## Non-Approval Boundary

This queue supports template improvement planning only. It does not overwrite source files, approve content, appoint leaders, move restricted data, broaden sharing, publish outputs, approve a website build, send messages, approve safety/legal/medical/theological decisions or close launch gates.
