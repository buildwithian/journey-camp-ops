# Organizational Implementation Board

This guide mirrors the live `Organizational Implementation Board!A5:N10` tab in the Journey V3 Leader Delegation and Readiness Tracker. Use the companion CSV as the local recovery/import template if the live Sheet needs to be rebuilt: [organizational-implementation-board.csv](organizational-implementation-board.csv).

## Purpose

The board is the first live execution surface for the current organizational implementation stage. It routes six real-world work areas into meetings, evidence capture and accountable decision updates:

- Owner and governance completion.
- HR-001 through HR-007 reviewer routing and sign-off.
- One small real Journey pilot cycle.
- Restricted real-data replacement approval.
- Website access-model approval.
- Production website build and publication readiness.

It does not appoint leaders, approve reviewer decisions, run a pilot, move restricted data, approve website access, build, deploy, publish or close launch gates.

## Live Workbook Placement

- Live tab: `Organizational Implementation Board!A5:N10`
- Dashboard metric: `Dashboard!A27:C27`, which counts rows whose status is not `Decision recorded`.
- Tracker instructions: `Instructions!A35:B35` records the non-automation and non-approval boundary.
- Resource Library evidence: `Quality Checks!A62:E62` records the live board as complete, with owner/status dropdown validation preserved.

## Status Values

Use only the live status dropdown values:

- `Not started`
- `Ready to schedule`
- `In session`
- `Evidence pending`
- `Decision recorded`
- `Blocked`
- `Review required`

Change a row to `Decision recorded` only when the accountable human evidence is present in the linked source of truth.

## Implementation Rows

| ID | Objective area | Starting status | Accountable role | Human evidence needed |
| --- | --- | --- | --- | --- |
| OIB-001 | Appoint owners and complete governance fields | Not started | Journey Lead | Programme-owner confirmation, backup coverage, availability/training readiness and final decision-rights authority. |
| OIB-002 | Complete seven human-review gates | Not started | Oversight Lead | Qualified reviewer name, authority, decision, date and conditions for HR-001 through HR-007. |

| OIB-005 | Approve website access model | Review required | Oversight Lead | Selected leaders-only, partly public/protected, or public plus authenticated-tools access model, with area decisions. |
| OIB-006 | Build and publish production website | Blocked | Knowledge Management Lead | Access, authority, copyright, content, restricted-data and technical publication evidence accepted before any build or publication. |

## Operating Boundary

Keep names, contact details, participant information, medical data, financial data and private source evidence out of public or broadly shared files unless the relevant access and data-placement approvals are already recorded. The board is a routing and evidence-management aid; Drive and the live Sheets remain the source of truth.
