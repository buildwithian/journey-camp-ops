# Gate Evidence Checklist

## Purpose

Use this checklist to prove what must be true before any remaining launch gate is marked complete. It is designed for the Journey V3 Leader Delegation and Readiness Tracker and mirrors the six open gates without approving them.

## Current Live-state Note

The live `Gate Evidence Checklist` tab is populated with the 22 evidence rows from [gate-evidence-checklist.csv](gate-evidence-checklist.csv), linked into Dashboard row 18, Instructions row 18, Launch Gates evidence fields and Resource Library Index quality checks. Keep the CSV as the recovery/import template.

## Privacy Rule

Record role-based evidence, dates, decisions and links only. Do not enter participant data, medical details, private contact details, financial account details or source-derived names in this checklist.

## Status Values

- Needs evidence
- Needs reviewer assigned
- Needs owner completion
- Needs authority approval
- Needs pilot run
- Needs debrief
- Needs access decision
- Needs data owner approval
- Needs publication approval
- In review
- Changes required
- Accepted / complete
- Blocked

## Operating Sequence

1. Keep every evidence item pending until the accountable human owner supplies proof.
2. Update the listed target tracker range only after the reviewer or approver records a decision.
3. Close a launch gate only when all evidence items for that gate are `Accepted / complete`.
4. Keep the production website blocked until access, content, copyright and final authority evidence is complete.
