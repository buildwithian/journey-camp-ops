# Template Acceptance Checklist

## Purpose

Use this checklist after a Template Improvement Queue row has a real draft output. It records the evidence needed before an improved template can become a canonical candidate, replace links, appear in website modules or move into published use.

## Current Live-state Note

The live `Template Acceptance Checklist` tab is populated with the twelve rows from [template-acceptance-checklist.csv](template-acceptance-checklist.csv), linked into Resource Library Dashboard row 13, Instructions row 11 and Resource Library Index quality checks. Dashboard row 13 currently shows twelve template acceptances pending.

## Privacy And Approval Rule

Keep acceptance rows open until the listed reviewer or authority records evidence. Do not enter private names, contact details, participant details, medical information, financial information, consent records, private photographs, unpublished reviewer notes or source-derived restricted content in this checklist.

## Acceptance Status Values

- Needs draft output
- Ready for acceptance review
- Accepted as canonical
- Changes required
- Deferred
- Withheld

## Restricted-data Check Values

- No restricted data
- Synthetic examples only
- Real names require approval
- Data owner approval required
- Financial data restricted
- Private contacts excluded
- Safety review required
- Needs access decision

## Operating Sequence

1. Draft the improved output from the matching Template Improvement Queue row using synthetic examples only.
2. Add the draft link and mark the acceptance row `Ready for acceptance review` only when the reviewer, evidence and replacement target are clear.
3. Keep the accepted output link blank until the authority records acceptance evidence.
4. Move the row to `Accepted as canonical` only after evidence proves the output is safe, approved and correctly placed.
5. Update Resource Library links, website modules or Drive placements only after acceptance, and preserve source files until a separate duplicate-resolution authority approves cleanup.

## Non-Approval Boundary

This checklist supports acceptance review only. It does not overwrite source files, approve content, appoint leaders, move restricted data, broaden sharing, publish outputs, approve a website build, replace canonical links, send messages, approve safety/legal/medical/theological decisions or close launch gates.
