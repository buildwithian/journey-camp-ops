# Leadership Review Communications

## Purpose

Use this queue to prepare and track the role-only messages needed to move the remaining launch gates from prepared to human-executed.

## Operating Rule

This template does not send messages, store personal contact details, approve review outcomes, appoint leaders, authorize restricted-data migration, approve website access or publish the production website.

## Required Workflow

- Keep audience fields role-based until the authorized sender uses the approved communication channel.
- Attach or reference the relevant live tracker, packet, checklist or decision row before a message moves to `Ready to send`.
- Record only the communication status, target/send dates and response evidence link in this shared queue.
- Record reviewer names, owner names, authority decisions and approvals only in the approved tracker fields or restricted locations.
- Keep participant, medical, financial, private personal and contact details out of this queue.

## Columns

| Column | Use |
| --- | --- |
| Communication ID | Stable identifier for the outreach item. |
| Gate(s) | Launch gate or gates the message supports. |
| Review checkpoint | Linked schedule, agenda, output, HR or weekly checkpoint. |
| Audience role(s) | Role-only audience description, not private contact details. |
| Message purpose | Why the communication exists. |
| Channel | Email, WhatsApp, meeting invite, document comment or in-person briefing. |
| Template / message prompt | Plain-language prompt for the sender to draft from. |
| Required links / attachments | Tracker, packet or checklist references needed before sending. |
| Sender role | Accountable role responsible for sending or coordinating the message. |
| Communication status | Current outreach status. |
| Target send date | Planned send date once known. |
| Sent date | Actual send date once sent. |
| Response / evidence link | Link to response, decision, minutes or logged evidence. |
| Notes | Boundary notes, blockers or follow-up constraints. |

## Status Values

- `Draft needed`
- `Ready to send`
- `Sent - awaiting reply`
- `Reply received`
- `Action logged`
- `Blocked`
- `Not needed`

## Closure Checks

Before marking a row `Action logged`, confirm that the response or meeting output is recorded in the proper destination: `Leadership Review Outputs`, `Human Review`, `Owner Appointment Routing`, `Pilot Log`, `Decisions`, `Website Access Review` or the relevant restricted Drive location.
